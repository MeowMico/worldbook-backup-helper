# 世界书工作台

[English README](README.md)

[Français](README.fr.md)

世界书工作台是一个用于 SillyTavern 世界书的扩展，支持在酒馆里直接编辑、备份、对比、回溯和导出世界书版本。

它主要面向边跑角色、边修改世界观/人设/规则/设定条目的创作者和读者。你不需要每次改动前都手动导出 JSON 备份，可以直接在工作台里修改，并保留本地版本历史。

## 功能亮点

- 在酒馆内置工作台中直接编辑世界书。
- 手机和窄屏会使用独立的 `世界书`、`条目`、`编辑`、`历史` 页面，不影响桌面端原有的三栏工作区。
- 第一次打开某本世界书时，自动保存 `Origin / 原版` 快照。
- 支持给每次改动创建命名实验，记录改前和改后版本。
- 支持当前版本、上一版、实验改前/改后、历史版本之间的 Diff 对比。
- 支持回溯到原版、实验结果或任意历史版本。
- 支持关键词查找、定位、替换、全部替换、删除、全部删除。
- 支持多选条目，并批量启用、禁用、删除或一键复制到另一本世界书，同时保留条目内容和设置。
- 支持作者端 MVU InitVar presets：给角色开场和当前聊天开场 swipes 绑定变量初始化预设。
- 支持实验改名、实验备注和实验历史搜索。
- 支持导出单个实验/版本 JSON，也支持导出某本世界书的全部本地历史。
- 支持中英双语界面。
- 支持浅色和深色主题。

标准编辑流程只处理世界书/知识书。可选的 `MVU InitVar` 页只会在你使用该工作流时读取当前角色卡开场和当前聊天开场 swipes。

## 安装方式

在 SillyTavern 中安装第三方扩展：

```text
扩展 -> 安装扩展程序 -> 粘贴本仓库链接
```

仓库链接：

```text
https://github.com/MeowMico/worldbook-backup-helper
```

安装后，可以从酒馆的扩展菜单中打开 `Worldbook Workbench / 世界书工作台`。

## VS Code / Cursor 独立版

本仓库同时提供不依赖 SillyTavern 运行环境的 VS Code / Cursor 独立版。它可以直接打开世界书 JSON、批量编辑和跨文件复制条目，并预览世界书条目最终进入 system/user/assistant 消息的顺序。快照、实验、Diff 和 Restore 会保存在同目录的 `<世界书名>.wbh-history.json` 旁路文件中，不会向原世界书 JSON 添加工作台字段。

独立版支持 80 步会话内撤销和重做；连续输入会合并为一步，新增、复制、启停、删除、批量替换和 Apply JSON 会各自作为完整的一步。保存或 Restore 后由持久快照继续负责回溯。

独立版提供“跟随 VS Code/Cursor”“English”“简体中文”三种语言选项，默认跟随编辑器显示语言。中文编辑器区域设置会使用简体中文界面，其他语言默认回退英文；也可以在工作台顶部或编辑器设置中强制指定语言，切换时不会丢失未保存的编辑。

请根据编辑器使用对应的商店安装：

- Microsoft 官方 VS Code：[Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=MeowMico.worldbook-workbench-vscode)
- Cursor、VSCodium 及其他 Open VSX 客户端：[Open VSX](https://open-vsx.org/extension/meowmico/worldbook-workbench-vscode)

完整的独立版编辑、预览和历史流程请参阅[简体中文使用手册](packages/vscode-extension/USER_GUIDE.zh-CN.md)，也可以在工作台顶部点击 `?` 随时打开。

建议从商店安装，这样后续版本可以正常自动更新。手动安装 `.vsix` 更适合本地测试；在 Microsoft 官方 VS Code 中，这类安装默认不会自动跟随商店版本更新。

## 推荐使用流程

1. 打开 `世界书工作台`。
2. 选择一本世界书。
3. 点击 `开始`，给本次实验命名，例如 `删掉机械化表达`、`加强城市规则`。
4. 在工作台里编辑条目。
5. 点击 `保存`，把世界书写回 SillyTavern。
6. 用 `对比` 查看改前和改后的差异。
7. 按需要标记 `保留` 或 `放弃`，写实验备注，或回溯到旧版本。

`保存` 不是只保存到插件草稿箱。点击 `保存` 后，工作台会把修改后的世界书直接写回 SillyTavern 原生世界书数据，不需要再去原生世界书界面二次保存。保存前和保存后都会自动生成版本记录。

如果你没有在工作台里改，而是在 SillyTavern 原生世界书界面里改，改完后可以回到工作台点击 `完成` 来保存改后版本。

## 工作台编辑

`编辑` 页支持常见的 SillyTavern 世界书条目字段，包括：

- 标题/备忘
- 内容
- 主要关键字和可选过滤器
- 常驻、禁用、可选、向量化
- 插入位置、角色、深度、顺序、触发概率
- 递归、分组、扫描、触发器、角色/标签过滤、额外匹配来源
- 粘性、冷却、延迟、自动化 ID、锚点/Outlet

第一次打开某本世界书时，当前状态会自动保存为 `Origin / 原版`。后续编辑不会覆盖原版。

点击 `保存` 后，酒馆里的当前世界书会立即更新。如果保存后觉得不理想，可以从右侧历史栏回溯到 `Origin / 原版`、实验改前/改后版本，或任意历史版本。

## 查找、替换和删除

工作台支持在条目字段里精确搜索关键词，可以：

- 跳转到上一个/下一个匹配
- 自动选中匹配文本
- 替换当前匹配
- 全部替换
- 删除当前匹配
- 确认后删除全部匹配

查找相关的编辑会进入本地撤回栈，误删后可以用 `撤回`。

## 跨世界书复制条目

在 `编辑` 页里，可以用条目列表左侧的复选框选择一个或多个条目，然后点击 `复制到…`。

复制时会保留条目的内容、关键词、插入位置、角色、深度、顺序、开关、触发概率、分组、触发器、过滤器、额外匹配来源等支持的设置。复制到目标世界书时会重新生成 UID，不会覆盖目标世界书里已有的条目。

目标世界书会自动生成复制前和复制后的版本记录。如果复制错了，可以从历史栏回溯。

## MVU InitVar Presets

`MVU InitVar` 页会扫描当前角色卡的 `first_mes`、`alternate_greetings`，以及当前聊天开场 swipes。你可以创建 InitVar preset，给每个开场绑定 preset，并把选中的 preset 同步进禁用的 `[initvar]变量初始化勿开` 条目，供作者本地测试。

作者本地测试时，可以开启 `开场自动注入`。当前聊天仍停在开场消息、还没有用户消息时，滑到已绑定的开场后，工作台会只把匹配的 preset 写进唯一禁用的 `[initvar]变量初始化勿开` 世界书条目，并生成写入前/写入后的历史快照。如果 MVU bundle 已经暴露 `window.Mvu`，工作台还会请求 MVU 把 InitVar 重新加载进当前开场 swipe，方便作者不发消息也能测试当前开场。

发卡时，可以在 preset 和开场绑定都做好后点击 `复制玩家脚本`。它会复制一个独立的酒馆助手 / JS-Slash-Runner 角色脚本，里面已经嵌入 preset 映射。把它粘进角色脚本库后，它会随角色卡一起导出。玩家端脚本只在 0 楼开场运行，监听当前 swipe，维护当前角色世界书里的禁用 `[initvar]变量初始化勿开` 条目，并请求 MVU 把对应 preset 重新加载进当前开场 swipe。

这个工作流只会保留一个可见的禁用世界书条目：`[initvar]变量初始化勿开`。Preset 和开场映射会存进这个条目的扩展元数据里，所以普通世界书编辑器里不会再显示额外的 `[MVU_INIT_PRESET:...]` 或 `[MVU_INIT_MAP]` 条目。旧版工作台已经生成的这些条目，会在下次通过工作台保存时迁移进 `[initvar]` 元数据。工作台本体仍然是作者端工具；复制出来的角色脚本才是可选的玩家端 runtime。

## 实验

实验适合记录一次很小、很具体的改动。

`开始` 会保存改前版本，`保存` 或 `完成` 会保存改后版本。每个实验可以有：

- 实验名
- 实验备注
- 改前快照
- 改后快照
- 保留/放弃状态

实验名和实验备注都可以在右侧历史栏搜索。

## 导出

你可以导出：

- 单个实验 JSON
- 单个历史版本 JSON
- 当前世界书的全部本地历史

这适合整理创作文件夹，或分享某个具体修改版本。

## 存储和隐私

如果配套服务端插件可用，快照和实验记录会以 JSON 文件保存在 SillyTavern 的用户备份目录下：

```text
backups/worldbook-backup-helper/<世界书名>/
```

检测到服务端文件存储可用时，扩展会自动把浏览器 IndexedDB 里的旧历史复制到这个目录里。旧的浏览器副本不会被删除，会保留作兜底。

在纯扩展模式下，快照和实验记录仍会保存在当前浏览器的 IndexedDB 中，也就是本地浏览器数据里。

扩展通过 SillyTavern 的世界书 API 读取和保存世界书。`MVU InitVar` 页还会读取当前页面里的角色开场和当前聊天开场 swipes，用于绑定 preset。它不会读取 cookies、API key、浏览器资料夹或无关的本地文件。

## 兼容性

目前按 SillyTavern `1.17.0` 公开源码适配和测试。

项目使用 SillyTavern 第三方扩展的标准格式，根目录包含 `manifest.json`。

TauriTavern 也可以使用同一个仓库链接安装。扩展会自动检测 TauriTavern 宿主，并把工作台标记为全屏移动端/桌面 surface，以适配 TauriTavern 的视口和安全区域行为。

## 已知限制

- 纯扩展模式的历史记录保存在浏览器本地。如果在服务端文件迁移或手动导出前清理浏览器数据，本地历史可能丢失。
- 暂时还不支持导入外部实验历史包。
- 本扩展专注世界书/知识书，不处理角色卡描述编辑。
