# 世界书工作台使用手册

[English](USER_GUIDE.md)

Worldbook Workbench 可以在 VS Code 或 Cursor 中直接编辑从 SillyTavern 导出的世界书 JSON，并在不启动酒馆的情况下模拟世界书条目的触发和插入位置。

## 快速开始

1. 用 VS Code 或 Cursor 打开一个包含酒馆世界书 JSON 的文件夹。
2. 右键世界书文件，选择 **世界书工作台：打开世界书工作台**。
3. 在左栏选择条目，在中栏编辑内容和设置。
4. 点击 **保存**，把当前草稿写回原世界书。
5. 点击 **预览**，查看哪些条目会触发，以及它们进入提示词的顺序。

所有功能也可以从命令面板打开。点击工作台顶部的 `?` 按钮可以再次打开本手册。

## 文件会保存在哪里

| 文件 | 用途 | 导出世界书时会包含吗？ |
| --- | --- | --- |
| `<世界书名>.json` | 世界书本身 | 会 |
| `<世界书名>.wbh.json` | 预览消息和激活设置 | 不会 |
| `<世界书名>.wbh-history.json` | 快照和实验历史 | 不会 |

两个旁路文件会保存在世界书旁边。工作台不会向原世界书 JSON 添加自己的配置字段。

## 条目编辑器

左侧条目列表支持筛选、多选、启用、禁用、删除、复制，以及把条目复制到另一本世界书。点击一个条目后，中栏会显示可编辑字段。

### 状态

- **普通**是关键词触发条目，使用绿色指示灯。
- **常驻**只要启用就会始终激活，使用蓝色指示灯。
- **向量化**依赖兼容的 SillyTavern 向量运行环境。工作台可以显示和编辑它，但不会自行运行向量搜索。

**使用可选筛选**对应酒馆的次关键词和 Selective 逻辑，它与条目状态相互独立。

### 插入位置

- **位置**决定条目进入哪个提示词区域，例如角色定义之前、角色定义之后、作者注释、示例消息、指定深度或出口。
- **顺序**用于排列处在兼容插入区域中的条目。
- 选择**指定深度**后会显示**深度**和**角色**。
- 只有勾选**使用触发概率**时，概率数值才会参与判定。

### 高级条目设置

展开内容下方的高级设置，可以编辑酒馆兼容的条目级覆盖、递归、包含组、粘滞/冷却/延迟、角色筛选、生成类型筛选和额外匹配来源。

向量搜索、Automation ID 和外部 Outlet 等字段需要 SillyTavern 运行环境才会实际生效，但工作台会在编辑和导出时保留这些字段。

## 预览设置

**预览设置是一个本地测试环境。** 它不是角色卡中的 `scenario` 场景文本，也不会被导出到世界书。

它主要用来回答这些问题：

- 在这段聊天里，哪些关键词条目会触发？
- 某个条目为什么没有触发？
- 已激活条目最终会按照什么 system/user/assistant 顺序插入？
- 扫描深度、递归、分组、概率和插入策略会怎样改变结果？

### 聊天消息

可以按顺序添加 system、user 和 assistant 消息，用它们模拟最近的聊天记录。关键词匹配会根据**扫描深度**检查这些消息。

世界书内容中的 `{{user}}` 等宏变量应继续保留。预览设置不要求填写一个固定的用户名字。

### 角色卡

点击**导入角色卡**可以读取 JSON 或 PNG metadata 中的角色卡。角色字段可以参与预览匹配，角色卡内置世界书也可以单独启用或关闭。导入的角色卡只读，不会被工作台修改。

### 全局激活设置

这里用于模拟酒馆式世界书激活。**工作台默认值**适合作者不受预算限制地检查内容；**ST 1.18 默认值**提供一个更接近酒馆原生设置的比较起点。

默认情况下 token 数只用于显示。只有主动配置上下文比例或预算上限时，预算才会限制预览中的条目激活。

### 高级预览设置

- **预览生成类型**用于测试仅允许在普通、继续、扮演用户、重抽、重新生成或静默生成时触发的条目。
- **强制激活 ID**可以让依赖外部运行环境的条目暂时进入预览。
- **粘滞激活 ID**和**冷却中的 ID**用于模拟酒馆在多轮生成之间保存的状态。
- **预览种子**让概率和分组选择在重复预览时保持一致。

点击**保存预览设置**会把这些配置写入 `<世界书名>.wbh.json`。这不是使用预览的必要条件，尚未保存的预览设置也可以直接运行。

## 提示词预览

修改世界书或预览设置后，点击**预览**。

右栏会显示：

- 已触发和未触发的条目数量
- 全部条目、已激活世界书和完整时间线的 token 数
- 按插入顺序排列的 system/user/assistant 时间线
- 每个条目触发或未触发的原因
- 需要 SillyTavern 运行环境才能完成的行为提示

点击预览中的世界书条目，可以返回该条目的编辑界面。

时间线表示酒馆式的消息骨架，不代表某个 AI 服务商最终使用的 API 包装格式。

## 查找与替换

**查找与替换**页可以搜索标题、关键词和内容。选择要处理的字段；只有需要严格匹配英文字母大小写时才勾选**区分大小写**。

**全部替换**会修改所有匹配项；**删除匹配项**只删除找到的文字，不会删除整个条目。保存前，这两种操作都可以撤销。

## 撤销、重做与历史

顶部的撤销和重做最多保存当前工作台会话中的 80 步未保存编辑。连续在同一字段输入会合并为一步。保存或恢复后，会话内撤销栈会被清空。

**历史**页提供持久记录：

- **快照**保存当前世界书草稿。
- **新建实验**保存一个有名称的基线。
- **保存结果**记录修改后的结果，供 Diff 或恢复使用。
- **对比**使用 VS Code 原生 Diff 编辑器。
- **恢复**会先保护当前草稿，再载入所选版本。

历史保存在 `<世界书名>.wbh-history.json`，不会写入世界书。

## 原始 JSON

**JSON** 页可以编辑完整世界书文档。点击**应用 JSON**后，工作台会先验证内容，再把它载入结构化编辑器；随后点击**保存**才会写入磁盘。

应用 JSON 可以撤销。无效 JSON 不会被应用。工作台会保留未知字段，以及 `entries` 原本使用 object 或 array 的形态。

预览设置底部的高级 JSON 编辑器操作方式相同，但它只修改 `.wbh.json` 预览旁路文件。

## 保存与导出

- **保存**把当前世界书草稿写回最初打开的文件，并自动记录保存前后的历史。
- **导出 JSON**把干净的世界书 JSON 副本写入指定位置。
- **保存预览设置**只写入独立的 `.wbh.json` 文件。

## 语言

顶部语言菜单提供**跟随 VS Code/Cursor**、**English**和**简体中文**。默认跟随编辑器。切换语言会立即更新当前工作台，不会丢失尚未保存的修改。

## 常见问题

如果右键菜单中没有工作台命令，请先信任当前文件夹，运行 **Developer: Reload Window**，然后也可以尝试从命令面板打开。

本地安装 `.vsix` 测试包后，请关闭已经打开的工作台页签、重新加载编辑器窗口，再重新打开世界书。在 Microsoft VS Code 中，手动安装 VSIX 的扩展通常不会继续自动跟随商店版本更新。

世界书、角色卡、预览设置和历史文件全部在本地处理，扩展不会上传其中的内容。
